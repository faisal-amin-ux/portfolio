01 / Case Study 2025

Packely

Designing a SaaS platform that turns 2D packaging artwork into photorealistic 3D renders for e-commerce and retail compliance.

Client Packely
Timeline 8 weeks
Role Product Designer
Services UX Research, UI Design, Prototyping
02 / Problem Statement

The challenge

Packely is a SaaS platform that generates photorealistic 3D renders of product packaging from uploaded 2D artwork files. When I joined, the service operated entirely through email — clients sent packaging files, received manual quotes, and got renders back days later with no visibility into the process.

The core problem was the absence of a self-service platform. Users — primarily CPG brand managers, packaging designers, and e-commerce operators — needed a way to upload their artwork, configure render settings, select camera angles, pay, and receive renders without manual back-and-forth. The existing workflow was slow, opaque, and impossible to scale.

Key friction points: (1) no unified interface for uploading files and configuring renders; (2) a complex order form with 12+ fields and no guidance, causing high abandonment; (3) no way to preview or select specific product angles from the 38 predefined views; (4) no dashboard for tracking order status or reviewing completed renders with feedback; and (5) no integrated payment flow.

Key pain points

  • 100% manual email-based workflow
  • 68% drop-off at the order form
  • 0 order tracking visibility
  • 38 angles configured via text description
  • 3+ days average turnaround
03 / Process

Understanding the render workflow

I started by mapping the existing user journey. The typical flow involved: discovering Packely via landing page, emailing artwork files (PDF, AI, PSD for design files; PNG, JPEG, TIF, SVG for reference images), receiving a manual quote, describing desired angles in text, and waiting for renders via email with no status updates.

I conducted stakeholder interviews with the Packely team and five user interviews with brand managers and packaging designers. Key findings:

  • Users needed to support multiple use cases in a single order — GS1 standards for retail distribution, Amazon marketplace specs, and fully custom requirements with specific dimensions and formats
  • Packaging files came in mixed formats requiring separate handling: reference images (PNG, JPEG, TIF, SVG) and design artwork files (PDF, AI, PSD)
  • Users wanted precise control over camera angles — 38 predefined views organized as 12 horizontal angles, 12 from a 15% elevated view, 12 from a 15% lowered view, plus top and bottom — rather than requesting "all angles" and filtering later
  • Post-render feedback was handled via email threads, making version tracking and revision management impossible
  • Users needed to specify technical details like file format (JPG, PNG, TIFF, SVG), pixel dimensions, margin percentages, clipping paths, and shadows — but had no guidance on what each setting meant

I then created user flows for three core journeys: first-time signup and onboarding with use-case selection, the main order creation and submission flow with angle selection, and the project management dashboard for tracking renders and providing feedback. I also designed the login and password reset flows to ensure a complete authentication experience.

04 / Solution

A self-service render platform

I designed a complete product ecosystem spanning marketing, order creation, and project management. The solution breaks down into four connected surfaces:

Landing and conversion. The landing page was restructured to clearly communicate the value proposition — photorealistic 3D renders from packaging artwork in 48 hours, ready for Amazon, GS1, and e-commerce. I added a use-case selector (GS1 Standards, Amazon Marketplace, Custom Requirements) so visitors immediately understood compliance relevance. The page includes a streamlined contact form and clear CTAs to reduce friction for first-time inquiries.

Order creation and configuration. The order form was redesigned from a flat list of 12+ fields into a stepped, contextual flow. Users first select one or more use cases, then configure settings per use case. For GS1 and Amazon, settings are pre-configured automatically. For custom requirements, I exposed advanced fields: file format selector (JPG, PNG, TIFF, SVG), pixel dimensions, margin percentage, and toggles for clipping path and shadows. Each field includes a hover tooltip with a brief description and a link to documentation, reducing support burden.

Product angle selector. I designed an interactive angle selector that lets users browse 38 predefined camera angles using arrow controls. The angles are organized into four categories: horizontal (12 angles), 15% elevated view (12 angles), 15% lowered view (12 angles), and top/bottom views. Users can select multiple angles, see a live count of selected views, and review their selection before submission. This replaced the previous workflow of describing angles in text.

Dashboard and project management. The workspace dashboard provides a unified view of all render projects with status indicators (In Progress, Completed), GTIN tracking, and date metadata. Users can create workspaces, organize projects, and access a detailed product view for each render job. The render review screen includes a comments section for structured feedback on lighting and composition, share functionality, and batch download of all renders.

Authentication and payment. I designed a complete signup flow with use-case onboarding immediately after account creation to accelerate first order submission. The login flow includes email/password entry and password reset via email verification. The payment screen integrates card payment with an order summary showing products, GTINs, use case, and total pricing before final submission.

05 / Results

From email to self-service

The unified platform replaced the previously fragmented email-based workflow. By consolidating file upload, render configuration, angle selection, payment, and project tracking into a single interface, users no longer needed to send files via email or wait for manual quotes.

The stepped order form with contextual guidance and tooltips reduced confusion at the configuration stage. Users could now select use cases, upload mixed file types, and configure custom settings with inline help rather than guessing field requirements.

The angle selector gave users precise control over render outputs, eliminating the back-and-forth previously required to communicate camera angles via text descriptions. Users could browse 38 predefined views, select exactly what they needed, and see their selection summarized before submission.

The dashboard introduced order visibility for the first time, allowing users to track progress from submission through completion, and to leave structured feedback on renders directly in the platform rather than managing scattered email conversations.

Outcomes

  • 68% → 22% form abandonment reduction
  • 100% email workflow replaced
  • 38 angles selectable with preview
  • 100% order visibility via dashboard
  • 0 email threads for feedback
