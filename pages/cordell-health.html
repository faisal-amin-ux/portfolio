01 / Case Study 2026

Cordell Health

Redesigning occupational healthcare operations for a UK-based SaaS platform serving 200+ clinics.

Client Modalys
Timeline 6 months
Role Senior Product Designer
Services Research, UI Systems, Prototyping
02 / Overview

The problem

Modalys operated a legacy healthcare platform that clinics used to manage occupational health assessments, medical records, and compliance reporting. The interface had grown organically over five years, resulting in fragmented workflows, inconsistent UI patterns, and a steep learning curve for new staff.

Clinic administrators reported spending 20+ minutes per patient just navigating between modules. The design-to-development handoff was chaotic — no component library existed, and every screen was built from scratch.

Key metrics

  • 200+ clinics using the platform
  • 20+ min average task time
  • 0 design system components
  • 3 days average handoff time
03 / Research

Understanding the clinic floor

I conducted 12 contextual inquiries across three clinic types: corporate health centres, independent practices, and NHS-integrated facilities. Shadowing administrators revealed that the most critical workflow — booking a patient through assessment to report generation — involved 14 distinct screens with no clear progress indicator.

Key findings from stakeholder interviews:

  • Administrators relied on muscle memory rather than visual cues to navigate
  • Doctors needed quick-access panels for patient history during consultations
  • Compliance officers exported data weekly but the export flow was buried three levels deep
  • Every clinic customised their workflow differently, requiring flexible yet consistent patterns

Methods

Contextual inquiry Stakeholder interviews Journey mapping Heuristic evaluation Competitive analysis
04 / Solution

A system that scales

I designed a modular dashboard architecture built around progressive disclosure. The four core modules — Scheduling, Assessments, Medical Records, and Compliance — were restructured with a consistent left-hand navigation and contextual right-panel for quick actions.

The component library grew to 60+ reusable components, from simple buttons to complex data tables with inline editing. Every component was documented with usage guidelines, accessibility notes, and responsive behaviour.

Deliverables

  • Design system (60+ components)
  • Interactive prototypes
  • Accessibility audit (WCAG 2.1 AA)
  • Developer handoff documentation
  • User testing report
Scheduling module — week view with drag-and-drop appointment management
Assessment workflow — step-by-step progress with contextual patient data
05 / Results

Measured impact

After a phased rollout over three months, we saw measurable improvements across all key metrics. The most significant change was in task completion time — what previously took 20+ minutes now averaged under 8 minutes for experienced users.

The development team reported that handoff time dropped from 3 days to under 4 hours, thanks to the comprehensive component library and detailed prototypes in Figma.

Outcomes

  • 60% reduction in task time
  • 90% faster design handoff
  • 4.8/5 admin satisfaction score
  • 100% WCAG 2.1 AA compliance
