01 / Case Study 2025

Packely

Restructuring navigation and introducing accessible colour systems for a mid-market logistics platform.

Client Clyro
Timeline 2 weeks
Role Senior Product Designer
Services IA, UI Design, Accessibility
02 / Overview

The problem

Packely served mid-market e-commerce brands with a logistics dashboard that managed inventory, shipping, and returns. The platform had grown feature-rich but navigation-heavy — users needed to memorise nested menu paths to complete basic tasks like printing shipping labels or checking delivery exceptions.

The colour palette relied heavily on reds and greens for status indicators, which failed WCAG contrast requirements and created barriers for colour-blind warehouse operators. Clyro needed a rapid turnaround that would not disrupt the existing backend architecture.

Key metrics

  • 47% of users got lost in navigation
  • 3.2 avg. clicks to print label
  • 0% WCAG AA compliance
  • 12 nested menu levels
03 / Research

Two weeks, three methods

With a compressed timeline, I focused on high-impact, low-effort research. I analysed clickstream data from the past 90 days to identify the most frequent user paths, then ran five unmoderated usability tests on the current interface to pinpoint friction points.

Key findings:

  • 86% of daily tasks involved only 4 core features, yet they were buried under 3+ menu levels
  • Red/green status indicators were indistinguishable to 8% of male users (deuteranopia)
  • Warehouse staff used the dashboard on shared tablets — touch targets were too small at 32px
  • The search function was the most-used escape hatch, suggesting information architecture failure

Methods

Clickstream analysis Usability testing Heuristic evaluation Accessibility audit
04 / Solution

Navigation that stays out of the way

I restructured the information architecture from a nested tree to a flat, task-based model. The four daily tasks — Ship, Track, Inventory, Returns — became persistent top-level tabs. Everything else moved to a collapsible "More" drawer, reducing cognitive load without removing features.

The colour system was rebuilt from the ground up. Status indicators now use shape + colour + text label: a circle for pending, a checkmark for complete, and a triangle for attention. The palette passed WCAG 2.1 AA on all background combinations, including the dark mode variant.

Deliverables

  • Restructured IA map
  • Accessible colour system
  • High-fidelity prototypes
  • Design tokens (JSON)
  • Accessibility audit report
Shipping workflow — one-click label generation with status preview
Accessible status system — shape, colour, and text working together
05 / Results

Immediate improvement

Because the changes were confined to the frontend layer, Clyro deployed them within a week of handoff. The impact was visible immediately — average clicks to print a shipping label dropped from 3.2 to 1.4, and navigation-related support tickets fell by 62% in the first month.

The accessibility improvements were validated by a third-party audit that confirmed WCAG 2.1 AA compliance across all status indicators and interactive elements.

Outcomes

  • 56% fewer clicks per task
  • 62% drop in support tickets
  • 100% WCAG 2.1 AA pass
  • 1 week from handoff to deploy
